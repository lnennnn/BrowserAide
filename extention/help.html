BrowserAide

BrowserAide is a multimodal Chrome agent that turns natural-language instructions into real browser actions. It observes both the DOM structure and the page screenshot, plans each step with a vision-language model, and uses Hybrid control to combine structured element actions with visual actions. For login, CAPTCHA, payment, or sensitive confirmation, BrowserAide pauses and lets you resume after handling the step manually.

Three modes

Each mode fits a different kind of page, not a different level of complexity.

DOM

Faster

For pages with clear structure and consistent rules.

Best for
Stable elements, easy targeting.
Trait
Direct path, faster feedback.

Visual

More flexible

For interfaces driven by layout, rendering, and visuals.

Best for
Complex layouts, frequent change.
Trait
Broader reach, screen-dependent.

Hybrid

Default

Balances stability and adaptability for most tasks.

Best for
Changing pages, less manual choice.
Trait
Steadier, better for multi-step work.

How to use

BrowserAide runs from the Chrome side panel and talks to your configured VLM backend.

1. Start backendRun the FastAPI server locally or with Docker before starting a task.
2. Set modelOpen Settings > Model, then enter the API key, base URL, and model name.
3. Choose modeUse Hybrid for most tasks; switch to DOM for structured pages or Visual for custom interfaces.
4. Describe taskWrite the goal, constraints, and desired output in the task box.
5. Start runClick Start Task. BrowserAide observes the page, plans each step, and executes in Chrome.
6. Monitor progressWatch Execution, Thinking, Action, Result, and Details to understand what is happening.
7. Resume if neededWhen login, CAPTCHA, or confirmation is required, complete it manually and click Resume.
8. Reset sessionUse Reset session when you want a fresh task state or the page context changed too much.

What you confirm

Automation does not replace judgment. High-risk moments still belong to you.

Login and verification

Passwords, captchas, and 2FA usually need your input.

Payment and submission

Payment, deletion, approval, and checkout pause for confirmation.

Exception handling

If the page shows a popup or odd permission state, you can fix it and continue.
